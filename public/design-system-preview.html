Outreach · Design system v1.0

A small system for a lean tool.

Five features, one CSS file, no framework. Tokens first, components composed from them, layout that leaves room for data. Built for operators who live in tables — KVK search, directory crawls, customers, leads, cold-email campaigns.

01 — Foundations

Principles

The tool is used, not read. Prioritize signal over decoration; keep chrome quiet so data can speak. Every component is a composition of tokens — never a hardcoded value. Both themes get the same care.

Density with air

Dashboard base is 14px, tables 13px. Rows breathe with 12px padding, not 20 — an operator scans a hundred records in a sitting.

One accent

Cobalt #3B5BFF carries primary action, focus, and selection. Everything else lives in the ink scale. Semantic color is not the accent.

State reads at a glance

Status is a pill, severity is a stripe, selection is a fill. Never rely on color alone — every state has form.

Motion is a hint

120–280ms, ease-out. A drawer slides. A row highlights. Nothing bounces. prefers-reduced-motion zeroes everything.

Color

Neutrals are cool with a subtle blue bias — greys picked, not defaulted. The accent is a single cobalt used for primary actions and focus. Semantic hues (emerald/amber/rose) stay separate from the accent so status always reads as status.

Motion

Duration ladder is 80 / 120 / 180 / 280ms — hover, focus, in-place transitions, entering surfaces. Ease-out for entrances, ease-standard for movement, no bounces. Zero at prefers-reduced-motion.

--dur-instant · 80ms

Hover color change, pressed states.

--dur-fast · 120ms

Focus rings, tooltip appearance, chip toggles.

--dur-base · 180ms

Row highlight, filter apply, tab switch.

--dur-slow · 280ms

Drawer, modal, toast entrance.

02 — Components

One primary per view. Secondary for supporting actions, ghost for toolbar-level actions, danger only for destructive with a matching confirm. Icon buttons need a tooltip and an aria-label.

Empty states with CTA

Every zero state gets a purpose and a next action. No shrug icons or witty copy — a one-line explanation and two buttons at most.

Queue is idle

Add a directory URL and we'll crawl it in the background. First page usually returns in under a minute.

Skeleton loaders

Only for content that takes > 300ms. Match the shape of what's coming — three rows, one avatar, one title. Never leave them running past 4s; swap to an error state.

Filter chips

Active filter has a fill and its value inline. Inactive is outlined. "+ Add filter" is dashed. Chips are removable with the × affordance.

Inline validation timing

  • Validate on blur, not on every keystroke. Exception: password strength.
  • Clear the error the moment they start correcting it. Don't leave a red input while they type the fix.
  • Server errors get inline placement, not a toast. A toast for "email is invalid" is homework in a different room.

ARIA checklist

  • Icon-only buttons: aria-label.
  • Modal: role="dialog", aria-modal="true", aria-labelledby pointing at the title.
  • Drawer: role="dialog", focus trap while open, focus returns to trigger on close.
  • Toast stack: role="status" (info) / role="alert" (danger).
  • Sortable column: aria-sort="ascending|descending|none" on the <th>.
  • Selected row: aria-selected="true".
  • Invalid input: aria-invalid="true" + aria-describedby pointing at the error.
  • Async loading: aria-busy="true" on the region until done.

Data density guidance

Choose the display by how the operator will read it, not by how the data looks.

Table

Default. Use when rows have ≥ 5 attributes and the operator compares across them (leads, customers, crawl queue).

  • Row height 40px, cell padding 12px
  • Sticky header, sortable, bulk-select
  • Truncate long text with tooltip
Card grid

Use for campaigns and dashboards — 1–3 key metrics per item, visual comparison, browse-ability over scan-ability.

  • Min column 220px, gap 12
  • Metric first, delta second, meta third
List

Use for feeds and activity — time-ordered events with one primary line and secondary meta (crawl activity, campaign timeline).

  • Row height 44–52px
  • Timestamp right-aligned, mono
